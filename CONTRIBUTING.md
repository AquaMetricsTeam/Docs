## Pull Requests

- One feature per PR.
- Keep PRs small.
- Request at least one reviewer.
- Resolve all comments before merging.

---

## Coding Standards

- Follow Clean Code principles.
- Meaningful variable names.
- No commented dead code.
- Keep methods short.
- Write reusable components.

---
